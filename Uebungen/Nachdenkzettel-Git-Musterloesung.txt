    4. Selber( mehr Entscheidungskraft, eigene Verantwortung )


    5. Master: keine Merge-Konflikte
    Branches: einzelne Features getrennt voneinander entwickeln
Vorteile Br: klare Struktur und Übersicht über die zugehörigen Commits/History, arbeiten an
Features die noch nicht ausgereift sind können einfach deaktiviert werden, Master Code bleibt
clean, ermöglicht paralleles Arbeiten

Nachteile Br: Team muss sich an Workflow halten, Feature branch basiert evtl. nicht auf aktuellem
Master Entwicklungsstand, Branch auf Branch erfordert Übersicht und Durchblick
